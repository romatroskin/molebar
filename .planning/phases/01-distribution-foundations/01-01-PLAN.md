<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer Mac → GitHub.com | Initial commit + branch creation push the LICENSE/README/.gitignore + gh-pages over HTTPS/SSH; Push Protection (enabled in Task 1) is the server-side defense. |
| Public Internet → GitHub Pages | The gh-pages site is public; anyone can `curl` the appcast.xml (Plan 06's responsibility) — that's intentional. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-01 | Information Disclosure | Sparkle EdDSA private key accidentally committed | mitigate | `.gitignore` has explicit `*.key`, `*.p8`, `*.p12`, `*.pem`, `private/` globs (Task 2); Push Protection enabled at GitHub side (Task 1); .env files also globbed. |
| T-01-01-02 | Tampering | Forked PR exfiltrates secrets via workflow | mitigate | No release workflow exists yet (Plan 06 builds it; this plan only ensures the repo is set up). When Plan 06 lands, the workflow triggers ONLY on `push` tags (never `pull_request_target`) — see Plan 06's threat model. |
| T-01-01-03 | Repudiation | Future developer claims the project was never properly licensed | mitigate | LICENSE committed at `chore(01-01): bootstrap` commit; GitHub's licenseInfo metadata is set on repo creation (`mit`); both verified in Task 5. |
| T-01-01-04 | Information Disclosure | gh-pages repo content accidentally serves credentials | mitigate | Plan 06's release workflow uses `peaceiris/actions-gh-pages@v4` with `publish_dir` pointing at a temp directory containing ONLY `appcast.xml` (per Pitfall A6); the `keep_files=true` flag never deletes index.html created here. |
| T-01-01-05 | Information Disclosure | Push Protection bypassed by force-push | accept | Branch protection on `main` is not configured in Phase 1 (deferred to Phase 1.5 per VALIDATION.md). For solo developer, the risk is low (only one push hand). Phase 1.5 hardens. |

</threat_model>
