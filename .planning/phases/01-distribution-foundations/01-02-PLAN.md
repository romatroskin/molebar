       /// Observes Sparkle's `canCheckForUpdates` so the button auto-disables
       /// while a check is in flight. Pattern from Sparkle's programmatic-setup docs.
       final class CheckForUpdatesViewModel: ObservableObject {
           @Published var canCheckForUpdates = false

               updater.publisher(for: \.canCheckForUpdates)
                   .assign(to: &$canCheckForUpdates)

       struct CheckForUpdatesView: View {

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | Tampering | Bundle ID conflicts with another app | mitigate | `app.molebar.MoleBar` is reverse-DNS based on a hypothetical `molebar.app` domain (D-03); collision risk is negligible. Locked in Info.plist (Task 1) and project.pbxproj (Task 1 via xcodegen). |

Out-of-order execution risk: low (Wave 2 + 3 sequencing enforced by depends_on). |
